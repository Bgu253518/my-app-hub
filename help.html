6. 📊 Word 报告上数与校验工作台 v1.0

审计 Word 附注报告自动填数工具。从 .docx 文件提取表格（或直接粘贴 Word 表格），配置科目列和数值列，导入审定 TB Excel 作为数据源，自动按科目名称匹配并填入数值。填完后自动检查表格内所有"合计"/"小计"行的加总正确性，导出填好的 Word 报告和加计校验 Excel。所有处理在浏览器本地完成，通过 mammoth.js + docx.js 实现 Word 格式读写。

三步操作流程

步骤一：加载与配置

  1. 上传 .docx：mammoth.js 提取文档内所有表格，选择目标表格
  2. 或粘贴表格：从 Word 复制表格直接粘贴到文本框，解析为 HTML 表格
  3. 配置列：单击表头标记"科目名称列"（蓝色标识），双击表头标记"待填数值列"
  4. 自动识别合计行：基于关键词"合计""小计""Total""Subtotal"自动标记，浅黄背景显示

步骤二：匹配与上数

  1. 加载数据源：上传审定 TB 或科目余额表 Excel，指定科目名称列和数据值列
  2. 自动匹配：去空格、统一全半角后按科目名称精确匹配；未匹配科目提供模糊匹配 Top 3 建议
  3. 手动拖拽配对：可将未匹配科目手动关联到 TB 行
  4. 保存映射模板：映射关系可存至 localStorage，下次对同客户直接加载
  5. 执行上数：遍历数据行（跳过表头/合计行），读取科目名称→查映射→填入数值到空单元格。已填单元格浅绿标识（.filled），跳过单元格灰色（.skipped），未匹配科目行浅红（.unmatched）

步骤三：校验与导出

  1. 加计检查：对每个合计行，向上寻找连续明细行，逐数值列求和后与合计行原值对比。差异 > 0.01 标记为"不符"→ 红底闪烁（diff-flash 动画）
  2. 差异列表：右侧展示每处差异的"行号、列名、原值、计算值、差异额"
  3. 导出 Word：docx.js 生成填好数的 Word 报告 (.docx)
  4. 导出 Excel：SheetJS 生成加计校验报告 (.xlsx)，含"校验报告"和"差异清单"两个 Sheet
  5. 复制备选：一键复制整个表格到剪贴板
💡 技巧：首次使用点击"加载演示数据"体验完整流程；模糊匹配阈值 15%，建议人工确认匹配结果；合计行识别后可手动增删标记。

7. 📋 BKD 底稿滚存与上数助手 v1.0

审计 BKD 底稿年度滚存工具。上传上年度底稿 Excel（含多个 Sheet），自动正则扫描表头识别日期列（24.12.31、2024-12-31、2024/12/31、20241231 等格式），提取年份。按用户指定的滚存规则"源列→目标列"将期末数硬拷贝为下年期初数（死数），清空原列值（公式保留结构、死数直接清空），更新列标题至新年份。然后导入本期科目余额表，根据客户科目映射模板自动填入新列。严格保留公式结构，不会破坏底稿原有计算逻辑。

四个标签页详解

📁 模板管理

维护客户科目与标准科目的对应映射表。字段包括：客户科目代码、客户科目名称、标准科目代码、标准科目名称、借贷方向。支持手动增删改行、导入/导出映射 Excel、加载 12 条演示模板。所有模板存储于 localStorage 键 bkdTemplates。

🔄 滚存操作

  1. 上传底稿 Excel → 勾选要处理的 Sheet（默认全选）
  2. 配置表头行号（默认第1行，可改为第2行）
  3. 自动扫描日期列：展示 Sheet名/列标题/识别年份/列字母，可手动修正年份或勾选忽略
  4. 设置滚存规则：源列（期末数，如 24.12.31）→ 目标列（滚存到，如 23.12.31位置）→ 新列标题（如 25.12.31）
  5. 执行滚存：遍历勾选的 Sheet → 源列值→目标列（死数）→ 源列清空（有公式保留公式置 null / 死数清空）→ 更新列标题
  6. 结果展示：复制数量、清空数量、标题更新数量

📊 上数与校验

  1. 上传本期科目余额表 Excel
  2. 根据模板匹配标准科目，在底稿中定位对应行
  3. 若目标单元格无公式 → 填入期末余额（根据借贷方向调整正负号）
  4. 若目标单元格有公式 → 跳过保护
  5. 未匹配科目红色列表展示
  6. 自动校验借贷平衡 + 年初年末一致性

📤 导出

下载完整底稿 xlsx（公式保留+格式完整）、校验报告 xlsx、JSON 项目备份。

💡 技巧：滚存前建议备份原始底稿；日期列识别基于正则，如果表头格式特殊（如"24.1231"无分隔符）可手动添加；公式单元格在滚存时保留公式结构但值清空，上数时也会跳过。

  3. 导入导出：支持 Excel 导入/导出映射表
  4. 上传余额表：自动识别列头（科目代码、科目名称、期初余额、借方发生额、贷方发生额、期末余额、借贷方向），用户可下拉修正

② TB 配置

  1. 选项A：上传已有 TB 底稿 Excel，指定科目代码列和本期填数列（可多选），解析所有行
  2. 选项B：自动从标准科目生成极简 TB 表（科目代码、科目名称、期初数、本期数空列）
  3. TB 表以可编辑表格预览

③ 自动上数

  1. 遍历科目余额表每一行 → 根据科目代码在映射表查找标准科目代码 → 在 TB 表中定位标准科目所在行

④ 校验导出

💡 技巧：科目性质影响填入值的来源（期末余额 vs 发生额），务必在模板中正确设置；当借贷方向不同时工具自动处理正负号，但建议人工复核方向标记。

9. 🎯 审计抽凭助手 v1.0

审计抽样工具，支持单家公司和多家公司批量抽样双模式。内置 Mulberry32 Seeded PRNG（给定种子保证任意次重复运行得到完全一致的随机序列）。三种抽样方法：MUS（货币单位抽样，累计金额数轴+等距投射）、设定规模随机抽样（Fisher-Yates 洗牌）、未设定规模随机抽样（自动计算样本量）。高价值项目 100% 强制选取，重复命中自动备注。导出含抽样日志+样本清单 Sheet + 抽样参数 Sheet（批量模式还含汇总 Sheet + 每公司独立 Sheet + 全局参数 Sheet）。

抽样方法详解

MUS（货币单位抽样）

  1. 筛选金额：按金额取值方式（正值/负值/绝对值）筛选数据
  2. 高价值项目：若开启"超 PM 强制选取"，金额 ≥ 阈值的项目标记为"高价值项目"，100% 加入样本
  3. 计算间距：RF = 风险系数映射表（风险水平, 信赖控制）→ SI = 有效总体金额 / (PM / RF)
  4. 生成累计金额数轴：0 到总金额，每笔金额占一段区间
  5. 随机起点：用种子生成 0~SI 的随机起点 RS
  6. 等距投射：从 RS 开始，每隔 SI 取一点，点落在哪个累计区间即选中哪笔
  7. 重复命中：若某项目金额 > SI 可能被多次选中，样本清单只列一次，备注"重复命中 X 次"
  8. 合并输出：高价值项目 + MUS 选中项目去重合并，若同一项目被两种方式选中 → "高价值项目"+"同时被 MUS 命中"

设定规模随机抽样

读取所有数据行作为抽样框 → Fisher-Yates 洗牌算法（种子驱动）打乱行顺序 → 取前 N 个为样本 → 选样方式标记"随机抽样"。

未设定规模随机抽样

先计算样本量 N = ⌈总体项目数 × 基础比例(%) / 100 / RF⌉ → 再调用"设定规模随机抽样"逻辑传入 N。

风险系数映射表

风险水平信赖控制系数 (RF)风险水平信赖控制系数 (RF)
高否1.0高是1.5
中否1.5中是2.0
低否2.5低是3.0

系数表可在"全局设置→风险系数映射表"中自由编辑，存 localStorage 持久化。

  1. 上传含多 Tab 的 Excel → 自动解析，Sheet 名为默认公司名称，可手动修改
  2. 生成可编辑参数配置表（每行=一家公司）：抽样方法、PM、金额列、金额取值、风险水平、信赖控制、超PM、阈值、样本量
  3. 向下填充：一键将首行参数复制到所有行
  4. 每家公司独立子种子 = 全局种子 + 公司行号，确保可复现且互不干扰
  5. 实时进度条显示 {已处理}/{总数}

💡 技巧：全局种子保证审计可复现——同一组种子+同一组数据，任何时候运行结果完全一致，满足审计底稿可追溯要求；MUS 适合大金额差异性总体，随机抽样适合金额较均匀总体；批量模式建议先确认配置表所有参数正确再执行。

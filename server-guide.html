🔧 第一章：初次搭建教程

1.1 你需要什么

条件要求说明
操作系统Windows 10 或更高Mac/Linux 也可运行，需自行安装 Python
Python3.9+（你的电脑已有 3.14）服务器运行环境，server.py 的引擎
磁盘空间100MB 以上SQLite 数据库非常轻量，百万条记录仅几十 MB
网络同一局域网（WiFi）团队成员需要在同一个路由器下

1.2 第一步：启动服务器

1
双击 start-server.bat

文件位置在 C:\Users\86152\my-app-hub\start-server.bat（或你 clone 的文件夹）。首次运行会自动安装 Flask 依赖（约 5-10 秒）。

2
查看启动信息

终端会显示：

=======================================================
  GDC 审计工时系统 — 本地服务器已启动
  数据库: .../gdc_data.db
  本机访问: http://127.0.0.1:5000
  团队访问: http://192.168.1.xxx:5000
  按 Ctrl+C 停止服务器
=======================================================

记下「团队访问」那个地址（如 http://192.168.0.199:5000），这是给团队成员用的。

3
验证服务器运行

打开浏览器，访问 http://127.0.0.1:5000/api/health，如果看到 {"status":"ok",...} 说明服务器正常。

✅ 此时服务器已就绪 你的电脑现在是一台小型数据库服务器，同 WiFi 下的人都能访问。

1.3 第二步：切换 GDC 系统到团队模式

4
打开 GDC 系统页面

访问 https://bgu253518.github.io/my-app-hub/gdc-task-system.html

5
切换到团队模式

点击页面右上角的 「本地模式」 按钮 → 切换为 「团队模式」。旁边的指示灯会先显示「检测中...」，连接成功后变为 🟢 「已连接」。

6
开始使用

创建项目、添加任务、填写工时——所有操作都会实时写入你电脑上的 SQLite 数据库。团队成员在你的局域网内打开同一网页、切到团队模式，就能看到共享数据。

⚠ 注意：团队成员需要先配置服务器地址 目前默认 API 地址是 http://127.0.0.1:5000（本机）。团队成员需要在浏览器 F12 Console 中执行：
API_BASE = 'http://192.168.x.xxx:5000'（替换为你的局域网 IP），然后刷新页面并切到团队模式。

1.4 团队成员连接步骤（简明版）

  1. 确保和你连接同一个 WiFi
  2. 打开 GDC 系统页面
  3. F12 打开控制台，输入 API_BASE = 'http://你的IP:5000'
  4. 点击右上角切换到「团队模式」
  5. 指示灯变绿 → 连接成功，开始使用

🔐 第二章：安全指引

2.1 数据存储安全

安全维度实际情况风险等级
数据存储位置gdc_data.db 文件在你的电脑上🟢 安全
数据加密SQLite 文件本身未加密（明文存储）🟡 中等
第三方访问数据不经过任何外部服务器🟢 安全
传输加密局域网内 HTTP 明文传输（无 HTTPS）🟡 中等
访问控制无登录认证，局域网内任何人可访问🔴 需关注
审计日志暂无操作日志记录🟡 中等
🟡 请注意 当前版本定位为「试用/测试阶段」。数据在局域网内是明文传输的，同一 WiFi 下的设备理论上可以访问。正式使用前建议由 IT 部门加固。

2.2 网络安全建议

🛡️ 安全最佳实践
  1. 只在局域网内使用——不要将 5000 端口暴露到公网
  2. 使用公司内网——公司 WiFi 通常有 WPA2/WPA3 加密
  3. Windows 防火墙已自动保护——首次启动时 Windows 会询问是否允许 Python 通过防火墙，点击「允许」即可
  4. 定期备份数据库——复制 gdc_data.db 到安全位置
  5. 不要让服务器长期无人值守运行

2.3 数据备份策略

📋
手动备份（推荐每周执行）

复制 gdc_data.db 文件到备份文件夹（如 D:\备份\），按日期重命名：gdc_data_2026-05-17.db

🔄
自动备份脚本

创建一个 backup.bat：

@echo off
set BACKUP_DIR=D:\GDC_Backup
if not exist "%BACKUP_DIR%" mkdir "%BACKUP_DIR%"
copy "C:\Users\86152\my-app-hub\gdc_data.db" "%BACKUP_DIR%\gdc_%date:~0,10%.db"
echo 备份完成: %BACKUP_DIR%\gdc_%date:~0,10%.db
pause

2.4 权限控制（当前阶段）

当前版本没有登录系统，权限控制依赖物理网络隔离：

  • 只有连上你 WiFi 的人能访问
  • 公司 WiFi 通常有密码保护
  • 下班后关闭服务器（Ctrl+C），数据绝对不会被访问
🔜 后续可加（移交 IT 后） 用户名密码登录、操作权限分级（只读/编辑/管理员）、操作日志审计、HTTPS 加密传输。

🛠️ 第三章：日常维护教程

3.1 启动与停止

操作方法说明
🚀 启动双击 start-server.bat终端窗口弹出即表示服务器运行中
🛑 停止按 Ctrl+C 或直接关掉终端窗口SQLite 支持 WAL 模式，强制关闭不会损坏数据
🔄 重启先 Ctrl+C 停止，再双击 start-server.bat适用于代码更新后或修改了配置

3.2 日常检查清单

  1. 每天早上启动服务器
  2. 打开 http://127.0.0.1:5000/api/health 确认运行正常
  3. 打开 GDC 网页，切换团队模式，确认指示灯绿色
  4. 下班前检查是否有未保存的操作
  5. 每周备份一次 gdc_data.db

3.3 故障排查

问题可能原因解决方法
双击 start-server.bat 闪退Python 未安装或不在 PATH打开 CMD 输入 python --version 检查
启动报错 "Address already in use"5000 端口被占用之前已启动了一个实例，检查是否有未关闭的终端窗口
指示灯红色「未连接」服务器未启动或 IP 不对检查 start-server.bat 是否在运行
成员连不上不在同一局域网 / 防火墙拦截检查是否同 WiFi；Windows 防火墙弹出提示时点「允许」
数据显示不一致浏览器缓存切换团队模式后按 F5 刷新
数据库文件越来越大正常现象SQLite 有自动回收机制，通常不需要手动清理

3.4 数据迁移

📦 迁移到新电脑
  1. 复制 gdc_data.db + server.py + start-server.bat 到新电脑同一文件夹
  2. 在新电脑安装 Python 3.9+
  3. 双击 start-server.bat（会自动安装依赖）
  4. 完成——所有历史数据都在
📤 迁移到正式数据库（MySQL/PostgreSQL）

SQLite 数据库文件可直接用工具导出为 CSV 或 SQL 脚本，IT 部门可据此导入到任何关系型数据库。gdc_data.db 包含 4 张表（projects/tasks/time_entries/allocation_rules），结构清晰。

3.5 升级服务器代码

当 GitHub 上有更新时：

  1. 停止服务器（Ctrl+C）
  2. 下载最新的 server.py 覆盖原文件
  3. 重新启动服务器
  4. 数据库文件 gdc_data.db 不受影响，数据不会丢失

❓ 第四章：常见问题 FAQ

Q: 我的数据会被别人看到吗？

🔒 不会。数据存在你电脑的 gdc_data.db 文件中。只有连上你同一局域网且知道 IP 的人才能访问。你下班关掉服务器后，任何人都无法访问。

Q: 我电脑关了，团队成员还能用吗？

❌ 不能。服务器在你电脑上，你电脑关机 = 服务器停止 = 团队模式不可用。不过成员可以切回「本地模式」继续使用 localStorage 存储的个人数据。

Q: 数据库文件丢了怎么办？

⚠ 如果没做备份，数据无法恢复。这就是为什么建议每周备份一次 gdc_data.db。

Q: 能同时多少人使用？

SQLite 支持多个读操作并发，但写入是串行的。对于 5-15 人的审计团队日常使用完全够用。如果超过 30 人同时频繁写入，建议升级到 PostgreSQL。

Q: 团队模式下需要联网吗？

需要局域网连接（WiFi），但不需要互联网。所有数据在局域网内传输，不经过外网。

Q: 本地模式和团队模式的数据互通吗？

❌ 不互通。本地模式数据在浏览器 localStorage 中；团队模式数据在服务器的 gdc_data.db 中。它们是两套独立的数据存储。切换模式后看到的是不同数据。

当你决定将这套系统正式推广，以下是 IT 部门需要的信息：

5.1 技术栈总览

组件技术文件

5.2 IT 部门可做的加固

  1. 添加认证——使用 Flask-Login 或 JWT 实现用户名密码登录
  2. 升级数据库——将 SQLite 替换为 PostgreSQL/MySQL（server.py 只需改一行连接配置）
  3. HTTPS——部署 Nginx 反向代理 + SSL 证书
  4. 部署到服务器——将 server.py 部署到公司内网服务器，保持 7×24 运行
  5. 操作日志——在 API 层增加操作审计日志
  6. 权限分级——实现 RBAC 权限模型（管理员/组长/成员）
  7. 定期自动备份——用 Windows 计划任务或 cron job 定时备份数据库

5.3 数据库表结构（供 IT 参考）

-- 项目表
CREATE TABLE projects (
    id TEXT PRIMARY KEY, code TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL, client TEXT DEFAULT '',
    created_at TEXT DEFAULT ''
);

-- 任务表（关联项目，级联删除）
CREATE TABLE tasks (
    id TEXT PRIMARY KEY, name TEXT NOT NULL, type TEXT NOT NULL,
    account TEXT DEFAULT '', unique_code TEXT NOT NULL UNIQUE,
    project_id TEXT REFERENCES projects(id) ON DELETE CASCADE
);

-- 工时记录表（关联任务，级联删除）
CREATE TABLE time_entries (
    id TEXT PRIMARY KEY, task_id TEXT NOT NULL,
    member TEXT NOT NULL, date TEXT NOT NULL,
    hours REAL NOT NULL, notes TEXT DEFAULT '',
    FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE
);

-- 分摊规则表
CREATE TABLE allocation_rules (
    id TEXT PRIMARY KEY, task_id TEXT NOT NULL,
    project_id TEXT NOT NULL, percent REAL NOT NULL,
    FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);
